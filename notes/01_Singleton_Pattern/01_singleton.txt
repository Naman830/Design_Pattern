Simple Mean:
The Singleton Design Pattern creates an object only once and reuses the same object throughout the entire application.

For example, when creating a connection between your application and the database, you create the connection only once and then use that same connection in different parts of the application.

Defination:
A Singleton ensures that a class has only one instance and provides a global access point to that instance.

Rules of the Singleton Pattern

1. The constructor must always be private.
2. The object is created through a static method (such as getInstance()).
3. A private field is used to store the single instance of the object.
